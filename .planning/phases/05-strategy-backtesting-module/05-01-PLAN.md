**File 1: backend/app/services/expression_parser.py**

Implement an AST-based expression parser using Python's `ast` module (stdlib). Per D-01 through D-04:

1. SAFE_FIELDS mapping (D-02):
   ```python
   SAFE_FIELDS = {
       'OPEN': 'open', 'HIGH': 'high', 'LOW': 'low', 'CLOSE': 'close',
       'VOL': 'vol', 'AMOUNT': 'amount', 'PRE_CLOSE': 'pre_close', 'CHANGE_PCT': 'change_pct',
   }
   ```

2. SAFE_FUNCTIONS dict (D-03) with implementations using pandas:
   - `MA(series, period)`: `series.rolling(window=int(period), min_periods=1).mean()`
   - `EMA(series, period)`: `series.ewm(span=int(period), adjust=False).mean()`
   - `STD(series, period)`: `series.rolling(window=int(period), min_periods=1).std()`
   - `MAX(series, period)`: `series.rolling(window=int(period), min_periods=1).max()`
   - `MIN(series, period)`: `series.rolling(window=int(period), min_periods=1).min()`
   - `REF(series, n)`: `series.shift(int(n))`
   - `CROSS(a, b)`: `(a.shift(1) <= b.shift(1)) & (a > b)` -- returns Boolean Series

3. SAFE_NODE_TYPES whitelist:
   `ast.Expression, ast.BinOp, ast.UnaryOp, ast.Call, ast.Name, ast.Constant, ast.Add, ast.Sub, ast.Mult, ast.Div, ast.UAdd, ast.USub, ast.Load`

4. `parse_and_validate(expr_str: str) -> tuple[ast.Expression, list[str]]`:
   - Parse with `ast.parse(expr_str, mode='eval')`
   - Walk AST recursively, validate each node type is in SAFE_NODE_TYPES
   - For `ast.Name` nodes: check id is in SAFE_FIELDS or function names (MA, EMA, STD, MAX, MIN, REF, CROSS)
   - For `ast.Call` nodes: validate func is ast.Name, check argument count (all functions require exactly 2 args), reject keyword args
   - Collect list of used field names
   - Raise SyntaxError for malformed expressions, ValueError for disallowed operations/unknown names
   - Return (validated_tree, used_fields_list)

5. `evaluate_expression(tree: ast.Expression, df: pd.DataFrame) -> pd.Series`:
   - Build namespace: map SAFE_FIELDS names to df columns, add SAFE_FUNCTIONS
   - Use `eval(compile(tree, '<expr>', 'eval'), {"__builtins__": {}}, namespace)`
   - Return resulting pandas Series

6. `evaluate_expression_with_custom(tree: ast.Expression, df: pd.DataFrame, custom_indicators: dict[str, pd.Series]) -> pd.Series`:
   - Same as evaluate_expression but also adds custom_indicators to namespace
   - This enables cross-indicator references like `EMA(MACD_DIF, 9)`

2. `async def run_backtest(self, ts_code: str, start_date: str, end_date: str, initial_capital: float, indicators_config: list[dict], buy_conditions: dict, sell_conditions: dict, stock_name: str = "") -> dict`:
   - Fetch DailyBar data from DB for ts_code within date range, sorted by trade_date ASC
   - Convert to pandas DataFrame with columns: trade_date, open, high, low, close, vol, amount, pre_close, change_pct
   - Evaluate all custom indicators using expression_parser, in dependency order:
     a. Build dependency graph from indicator expressions (scan for references to other indicator names)
     b. Topological sort to determine evaluation order
     c. Evaluate each indicator, adding its Series to the DataFrame for subsequent indicators
     d. Detect and reject circular dependencies with clear error message
   - Run daily iteration loop (synchronous, CPU-bound):
     a. For each trading day i:
        - Build row_values dict with all base fields + all custom indicator values at index i
        - Check sell conditions FIRST: only sell positions where buy_date != current_date (T+1 rule per Phase 4 pattern)
        - If sell triggered: sell ALL sellable shares at close price, calculate fees (commission 0.025%, stamp tax 0.1%), add to cash
        - Check buy conditions: only buy if currently no positions (full position per open question resolution)
        - If buy triggered: calculate shares = int(cash * 0.95 / close / 100) * 100 (95% of cash, rounded to board lot), calculate commission
        - Record equity: cash + market_value of any open positions
   - Calculate buy-and-hold baseline: baseline_worth = initial_capital * (close_T / close_0) for each day
   - Calculate all 8 statistics (D-14): total_return_pct, annualized_return_pct, max_drawdown_pct, trade_count, win_rate_pct, profit_factor, sharpe_ratio, avg_holding_days
   - Save BacktestSession and BacktestTrade records to DB
   - Return dict: {session_id, trades, equity_curve, baseline_curve, statistics}

3. Fee calculation (extract logic inline, same as PracticeService):
   - commission = round(amount * 0.00025, 2)
   - stamp_tax = round(amount * 0.001, 2) on sells only
   - shares = int(cash_available * 0.95 / close / 100) * 100

4. Statistics calculation methods (private):
   - `_calc_statistics(trades, equity_curve, initial_capital, start_date, end_date) -> dict`:
     - total_return_pct = (final_equity - initial_capital) / initial_capital * 100
     - annualized_return_pct = total_return_pct * (365 / total_calendar_days) -- simple annualization
     - max_drawdown_pct: track running max of equity curve, compute (current - running_max) / running_max, take minimum
     - trade_count: count of completed buy-sell pairs
     - win_rate_pct: pairs with profit > 0 / total_pairs * 100
     - profit_factor: sum(winning_profits) / abs(sum(losing_losses)) -- handle zero denominator
     - sharpe_ratio: mean(daily_returns) / std(daily_returns) * sqrt(252) -- annualized, handle zero std
     - avg_holding_days: mean(holding_days across all pairs)
   - `_evaluate_condition_tree(tree, row_values, indicator_series, idx) -> bool`: recursive evaluator
     - For group nodes: AND = all children true, OR = any child true
     - For rule nodes: evaluate operator per D-07:
       * `>`, `<`, `>=`, `<=`: standard comparison on row_values
       * `==`: abs(left - right) < 1e-9
       * `golden_cross`: prev_left <= prev_right AND curr_left > curr_right (check NaN, idx >= 1)
       * `death_cross`: prev_left >= prev_right AND curr_left < curr_right
       * `break_above`: curr_left > right_val AND prev_left <= right_val
       * `break_below`: curr_left < right_val AND prev_left >= right_val
       * `enter_overbought`: curr_left > right_val AND prev_left <= right_val
       * `enter_oversold`: curr_left < right_val AND prev_left >= right_val
     - All operators: if either value is NaN, return False
