    **Modify frontend/src/components/KLineChart.vue** to add signal and pattern markers.

    The approach: Add separate scatter series for signal markers and pattern markers on the K-line chart.

    **Signal markers implementation (per D-06, D-07):**

    Build signal marker data from `stockStore.vpaData.signals`. Create a lookup from trade_date to signal info:
    ```javascript
    const signalMap = new Map(
      (stockStore.vpaData.signals || []).map(s => [s.trade_date, s])
    )
    ```

    Add two scatter series for signals (one for confirmation, one for anomaly):
    - **Confirmation signals** (volume_up_rise, volume_down_fall):
      - Scatter series on xAxisIndex:0, yAxisIndex:0
      - Data: for each date index where signalMap has a confirmation signal, position at [index, high_value + offset] for up signals or [index, low_value - offset] for down signals
      - Symbol: `'triangle'` for up (size 8), rotated 180deg for down (size 8)
      - Color: '#26a69a' for up (green per A-share convention), '#ef5350' for down (red)
      - Only shown when `chartStore.showSignals` is true (check in computed, return empty array if false)
    - **Anomaly signals** (long_candle_low_volume, short_candle_high_volume, rising_volume_decline):
      - Scatter series on xAxisIndex:0, yAxisIndex:0
      - Symbol: `'diamond'` (size 10) for visual distinction per D-07
      - Color: '#ffeb3b' (yellow) -- stands out as a warning
      - Position: above the K-line bar (high + offset)
      - Only shown when `chartStore.showSignals` is true

    **K-line pattern markers implementation (per D-08):**

    Build pattern marker data from `stockStore.vpaData.patterns`:
    ```javascript
    const patternMap = new Map(
      (stockStore.vpaData.patterns || []).map(p => [p.trade_date, p])
    )
    ```

    Add a scatter series for pattern markers:
    - Scatter series on xAxisIndex:0, yAxisIndex:0
    - Data: for each date index where patternMap has a pattern, position at [index, high_value + offset]
    - Symbol: `'circle'` (size 6) per D-08
    - Color: '#d1d4dc' (light gray, visible on dark background)
    - Only shown when `chartStore.showPatterns` is true

    **Tooltip enhancement:**

    Extend the existing tooltip formatter to show signal and pattern info. When a tooltip is triggered (axis type), check if the hovered date has signals or patterns:
    ```javascript
    // In the existing formatter function, after MA lines section:
    const dateStr = candleParam.axisValue
    const signal = signalMap.get(dateStr)
    const pattern = patternMap.get(dateStr)

    if (signal || pattern) {
      let extraInfo = '<div style="margin-top:4px;border-top:1px solid #363a45;padding-top:4px">'
      if (signal) {
        const signalColor = signal.signal_type.includes('long_candle') || signal.signal_type.includes('short_candle') ? '#ffeb3b' : (signal.direction === 'up' ? '#26a69a' : '#ef5350')
        extraInfo += `<div style="color:${signalColor}">信号: ${signal.description}</div>`
      }
      if (pattern) {
        extraInfo += `<div style="color:#d1d4dc">形态: ${pattern.name} - ${pattern.description}</div>`
      }
      extraInfo += '</div>'
      tooltipContent += extraInfo
    }
    ```

    **Data alignment:** Signals and patterns come as arrays keyed by trade_date (string format "YYYYMMDD"). The K-line dates array is also "YYYYMMDD" strings. Align by index lookup or Map.

    **Performance consideration:** Only build marker arrays when the corresponding toggle is on. When showSignals is false, the signal scatter series should have empty data array. When showPatterns is false, the pattern scatter series should have empty data array. This avoids unnecessary computation.

    **Important:** Do NOT modify existing chart functionality (candlestick, volume, MA, indicators). Only ADD new scatter series for markers and extend the tooltip formatter.
