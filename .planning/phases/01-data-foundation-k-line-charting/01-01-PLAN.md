    **API Endpoints:**

    1. **backend/app/api/stocks.py** -- Stock search endpoint:
       - Router with prefix `/stocks`
       - GET /api/stocks/search?keyword=xxx:
         - Validate keyword is not empty (400 if empty, per D-06 debounce is client-side)
         - Query Stock table: WHERE ts_code LIKE `%keyword%` OR name LIKE `%keyword%` OR symbol LIKE `%keyword%`
         - Return list of {ts_code, name, symbol} sorted by ts_code
         - Limit results to 20 rows
       - GET /api/stocks/{ts_code} -- Return single stock info (for chart header display)

    2. **backend/app/services/data_fetcher.py** -- Data fetch orchestrator:
       - Class `DataFetcher` with session and tushare_client dependencies
       - `ensure_stock_list(session)`: Check if Stock table has data. If empty, call tushare get_stock_list(), bulk insert all stocks. This is called on app startup.
       - `fetch_daily_data(session, ts_code, start_date=None, end_date=None)`:
         - Check DailyBar table for existing data for this ts_code
         - If data exists: return cached data filtered by date range if provided
         - If no data exists: fetch ALL historical data from tushare (per D-12, fetch full history on first request), insert into DailyBar table with dedup (INSERT OR IGNORE on unique constraint), then return
         - If partial data exists (last cached date < today): fetch incremental data from last_date to today, append to DB, then return
         - Sort results by trade_date ASC
         - Return format: list of {trade_date, open, high, low, close, vol, amount}
       - `get_cache_info(session, ts_code)`: Return {last_date, total_bars, last_updated} for display in frontend (per D-13 "display last update time")

    3. **backend/app/api/daily.py** -- Daily K-line data endpoint:
       - Router with prefix `/daily`
       - GET /api/daily/{ts_code}:
         - Validate ts_code format (should be 6 digits + . + 2 letters, e.g. "000001.SZ")
         - Use DataFetcher to get data (handles cache vs fetch)
         - Return {ts_code, name, data: [{trade_date, open, high, low, close, vol, amount}], cache_info: {last_date, total_bars}}
         - On tushare error: return 502 with {error: "数据获取失败", detail: msg, retry: true} (per D-15)
       - POST /api/daily/{ts_code}/refresh -- Force re-fetch from tushare, update cache
